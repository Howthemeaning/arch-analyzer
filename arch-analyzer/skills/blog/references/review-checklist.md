# 博客审查清单

## 结构审查

### 标题审查

- [ ] 标题是否吸引人且准确
- [ ] 标题是否使用动词或疑问句
- [ ] 标题是否避免过于技术化或过于通俗

### 结构完整性

- [ ] 是否有背景、问题、方案、总结四部分
- [ ] 各部分比例是否合理
- [ ] 是否有清晰的章节划分

### 开头审查

- [ ] 开头是否简明扼要
- [ ] 开头是否不超过200字
- [ ] 开头是否引出核心问题

### 结尾审查

- [ ] 结尾是否总结要点
- [ ] 结尾是否给出建议或展望
- [ ] 结尾是否不超过300字

## 内容审查

### 技术描述

- [ ] 技术描述是否准确
- [ ] 是否有具体代码位置引用
- [ ] 是否避免模糊表述

### 代码示例

- [ ] 代码示例是否正确
- [ ] 代码是否有注释
- [ ] 代码是否不超过30行
- [ ] 代码是否突出关键部分

### 问题分析

- [ ] 是否清晰描述技术难点
- [ ] 是否说明问题的复杂性
- [ ] 是否有改进建议

### 解决方案

- [ ] 是否有核心思路说明
- [ ] 是否有详细实现步骤
- [ ] 是否有具体代码示例

## 图表审查

### 图表数量

- [ ] 是否至少有1个 Mermaid 图表
- [ ] 图表数量是否适中（1-3个）

### 图表质量

- [ ] 图表代码是否正确嵌入
- [ ] 图表是否能正确渲染
- [ ] 图表是否有说明文字
- [ ] 图表是否简洁清晰

### 图表类型

- [ ] 图表类型是否适合内容
- [ ] 图表是否正确表达关系
- [ ] 图表节点命名是否清晰

## 语言审查

### 语言风格

- [ ] 是否使用简体中文
- [ ] 技术术语是否准确
- [ ] 是否避免口语化表达
- [ ] 是否保持专业但易懂

### 表述清晰

- [ ] 是否有模糊表述（如"某种方式"）
- [ ] 是否有过度简化（如"很简单"）
- [ ] 是否有无依据结论
- [ ] 是否流畅易读

### 技术术语

- [ ] 通用术语是否保留英文
- [ ] 有译名术语是否使用译名
- [ ] 新术语是否有解释

## 格式审查

### 文件命名

- [ ] 文件命名是否规范（YYYY-MM-DD-slug.md）
- [ ] slug 是否从标题生成
- [ ] slug 是否为英文、小写、连字符

### Markdown 格式

- [ ] Markdown 格式是否正确
- [ ] 标题层级是否正确
- [ ] 代码块是否正确标注语言

### 链接引用

- [ ] 是否有相关文件链接
- [ ] 链接是否正确
- [ ] 是否引用 PLAN.md
- [ ] 是否引用相关图表

## 字数审查

- [ ] 字数是否在 2000-5000 范围
- [ ] 字数是否适中（推荐 3000-4000）
- [ ] 各部分字数比例是否合理

## 评分标准

### 评分维度

| 维度 | 权重 | 评分范围 | 说明 |
|------|------|---------|------|
| 结构完整性 | 20% | 0-20 | 四部分完整、比例合理 |
| 技术准确性 | 30% | 0-30 | 描述准确、代码正确 |
| 图表质量 | 20% | 0-20 | 正确嵌入、清晰易懂 |
| 代码示例 | 15% | 0-15 | 准确、有注释、突出关键 |
| 语言流畅性 | 15% | 0-15 | 专业、易读、无模糊表述 |

### 评分等级

| 总分 | 状态 | 处理方式 |
|------|------|---------|
| >= 85 | 通过 | 可以发布 |
| 70-85 | 建议修订 | 根据审查意见修订 |
| < 70 | 必须修订 | 必须修订后重新审查 |

## 审查流程

### Step 1: 结构审查

检查博客的整体结构是否完整、比例是否合理。

### Step 2: 内容审查

检查技术描述是否准确、代码示例是否正确。

### Step 3: 图表审查

检查图表是否正确嵌入、是否能正确渲染。

### Step 4: 语言审查

检查语言风格是否专业、是否有模糊表述。

### Step 5: 格式审查

检查文件命名、Markdown 格式、链接引用。

### Step 6: 计算总分

汇总各维度评分，计算总分。

### Step 7: 决定处理方式

根据总分决定：通过、建议修订、必须修订。

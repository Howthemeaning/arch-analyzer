## 各部分详细说明

### 项目概览

简要介绍项目的基本信息：
- 项目名称：从 package.json、README 或目录名获取
- 技术栈：主要语言和框架
- 代码规模：文件数和代码行数
- 架构模式：识别的主要架构模式

### 技术架构

展示项目的整体架构：
- 架构模式描述：说明采用的架构模式及其特点
- 系统架构图：使用 Mermaid graph TB 展示
- 模块划分：列出主要模块及其职责
- 模块依赖图：使用 Mermaid graph LR 展示

### 目录组织

说明项目的目录结构：
- 目录树：使用 text 格式展示主要目录
- 职责划分：说明各目录的职责

### 功能分析

分析项目的功能：
- 核心功能：列出项目的主要功能
- 模块功能：详细说明各模块的功能

### 数据流

展示核心数据流转：
- 使用 Mermaid sequenceDiagram 展示
- 包含主要参与者和数据流向

### 技术难点

列出技术难点：
- 每个难点包含描述、位置、难度等级、博客潜力
- 难点等级：high（复杂实现）、medium（特殊处理）、low（常规问题）
- 博客潜力：high（适合写博客）、medium（可写）、low（不适合）

### 技术亮点

列出技术亮点：
- 创新方案、最佳实践、优秀设计

### 改进建议

给出改进建议：
- 技术债务清理
- 架构优化
- 代码质量提升
